The Agentic Cookbook

Multi-platform native development, constrained by recipes, built by agents

Cross-Platform Is Dead

The middleware era solved a human problem: writing native code for every platform is slow and error-prone.

The Middleware Tax

  • Write once, abstract everywhere
  • Lowest common denominator APIs
  • Platform-specific bugs in generic code
  • Always one version behind native

The Players

React Native Flutter Xamarin SwiftUI Kotlin Multiplatform Electron MAUI

All built on the same premise: humans can't write native code fast enough.

Multi-Platform Is Native

Agents don't need abstraction layers. Given the right constraints, they write native code for every platform — directly against the best API.

Cross-Platform (old) Your Code → Middleware → Platform API → Native UI Multi-Platform (new) Recipe → Agent → Platform API → Native UI → Platform API → Native UI → Platform API → Native UI

The "write once" isn't the code anymore — it's the constraints.

Constrain, Don't Abstract

Instead of abstracting platforms away, define what "trusted" means and let the agent figure out the implementation.

Principles

Universal truths. "Code must be accessible." "Errors must be recoverable." 14 dimensions of what trusted code means.

Guidelines

Topic-organized rules. Security, testing, UI, accessibility, networking. 88 guidelines agents must follow.

Compliance

Verification layer. 10 categories, 81 checks. Automated validation that the output meets the constraints.

Recipes: The New "Write Once"

A recipe defines what to build, not how. The agent writes platform-native code.

---
title: "Status Bar"
platforms: [ios, macos, web, kotlin, swift, typescript]
type: recipe
---

## Behavioral Requirements

- **slide-in-animation**: The bar MUST slide in from the
  bottom edge with an animated transition.
- **indeterminate-spinner**: MUST display a platform-native
  indeterminate progress spinner.
- **overlay-not-push**: MUST overlay content, not push it.
- **non-blocking**: MUST NOT block interaction beneath it.

## Appearance
- Height: 28–32pt
- Background: system blur or translucent fill

Same recipe → UIKit, AppKit, Jetpack Compose, React, vanilla HTML — no middleware

Three-Layer Architecture

Every recipe is grounded in guidelines, verified by compliance checks.

┌─────────────────────────────────────────────┐ │ Recipes (27) │ │ Define what to build for each platform │ ├─────────────────────────────────────────────┤ │ Compliance (81 checks) │ │ Verify the output meets constraints │ ├─────────────────────────────────────────────┤ │ Guidelines (88) │ │ Topic rules: security, a11y, UI, testing │ ├─────────────────────────────────────────────┤ │ Principles (18) │ │ Universal truths about trusted software │ └─────────────────────────────────────────────┘

The Scoping Problem

Agents can build. But what should they build?

Product discovery is still a human bottleneck.
Requirements are scattered across conversations, docs, and assumptions.
Agents need structured, comprehensive input to produce good output.

What if agents could help scope the product too?

The Agentic Interview Team

A multi-agent product discovery system. 18 specialists. Structured + exploratory questioning.

┌──────────────────────────────────────────────────┐ │ Meeting Leader (Skill) │ │ Orchestrates, talks to user, writes │ │ transcripts, maintains living checklist │ └──────┬──────────────┬──────────────┬─────────────┘ │ │ │ ┌──────▼──────┐ ┌─────▼──────┐ ┌────▼─────────┐ │ Transcript │ │ Specialist │ │ Specialist │ │ Analyzer │ │ Interviewer│ │ Analyst │ │ │ │ (per domain)│ │ (per domain) │ └─────────────┘ └────────────┘ └──────────────┘

Hub-and-spoke: leader orchestrates, specialists come and go as needed.

The Interview Flow

Structured Phase Cover known questions via checklist ↓ Explore Phase Specialists follow threads, probe implications ↓ New Knowns Discovered Analysis surfaces questions nobody thought to ask ↓ ↻ Back to structured → explore → repeat ↓ Session End Summary, resume anytime, living checklist persists

Two files per exchange: transcript (raw Q&A) + analysis (implications, gaps, new questions).

Using It

# 1. Set up
git clone agentic-cookbook && git clone agentic-interview-team

# 2. Install the cookbook into your project
cd my-app && /install-cookbook

# 3. Scope your product with the interview team
/interview
# → "What are you building? Tell me about yourself..."
# → 18 specialists ask the questions you'd miss
# → Timestamped transcripts + deep analysis

# 4. Plan and build
# Agent reads transcripts + cookbook recipes
# Writes native AppKit, UIKit, Jetpack Compose, React...
# Compliance checks verify the output
