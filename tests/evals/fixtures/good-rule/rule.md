# Test Rule

Prerequisite: Read and follow `authoring-ground-rules.md` before applying this rule.

This rule enforces clear commit messages on every commit.

---

## Requirements

1. **Write descriptive commit messages** — every commit message MUST describe the what and why of the change.
2. **Use imperative mood** — commit messages MUST start with a verb in imperative mood (e.g., "Add", "Fix", "Remove").
3. **Keep subject under 72 characters** — the first line of the commit message MUST NOT exceed 72 characters.

---

## MUST NOT

- Do not use generic messages like "fix" or "update" without context.
- Do not commit without a message.
- Do not amend published commits without team approval.
